## Section Content Guidelines

### Topic Overview
- Keep it brief (2-3 sentences)
- Answer: What is this? Why does it matter?
- Set context for the rest of the notes

### Remember Section
- Focus on memorizable content
- Use lists for easy scanning
- Include vocabulary with clear definitions
- Keep facts concise

### Understand Section
- Use simple, clear language
- Explain "what it means" not just "what it is"
- Use examples to clarify
- Avoid jargon or explain it

### Apply Section
- Connect to real life
- Show practical uses
- Include step-by-step procedures
- Make it actionable

### Analyze Section
- Break complex ideas into parts
- Use comparisons (tables work well)
- Show connections and relationships
- Identify patterns

### Evaluate Section
- Present balanced view (pros/cons)
- Include multiple perspectives
- Provide criteria for judgment
- Support with evidence

### Create Section
- Suggest original projects
- Encourage innovation
- Propose improvements
- Challenge thinking

### Practice Questions
- Include 2-3 questions per level
- Use appropriate question starters for each level
- Questions should test that specific cognitive level
- See blooms-taxonomy.md for question patterns

## Formatting Guidelines

**Use clear formatting:**
- H1 (`#`) for main title
- H2 (`##`) for Bloom's levels
- H3 (`###`) for subsections
- Bold (`**text**`) for key terms
- Bullet points for lists
- Tables for comparisons
- Horizontal rules (`---`) to separate major sections

**Keep it readable:**
- Short paragraphs (2-3 sentences)
- White space between sections
- Clear visual hierarchy
- Consistent formatting throughout

## Content Adaptation

**For web content:**
- Include source link
- Focus on main ideas from the page
- Adapt web formatting to note structure
- Include key points, not every detail

**For documents:**
- Reference the document
- Follow document's logical flow when possible
- Preserve important structure
- Capture essential content at each Bloom's level

**For topic titles only:**
- Research the topic if unfamiliar
- Use general knowledge
- Structure around standard aspects of the topic
- Acknowledge if more information would be beneficial
